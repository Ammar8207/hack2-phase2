# Todo App – Phase II Constitution

## Phase
Phase II – Full-Stack Web Application

## Non-Negotiable Rules
- All functionality must be generated via spec-driven development
- No manual application code is allowed
- Claude/Kiro must read specs before implementation
- Backend and frontend must enforce user isolation
- JWT authentication is mandatory

## Scope
This phase delivers a multi-user web-based Todo application with persistent storage.

## In Scope
- Task CRUD (Create, Read, Update, Delete, Complete)
- User authentication (signup/signin)
- REST API with JWT protection
- Next.js frontend
- FastAPI backend
- Neon PostgreSQL persistence

## Out of Scope
- AI chatbot
- MCP tools
- Kubernetes
- Kafka
